Some superficial changes was made to the frontpage. The company logo was changed to have a transparent background. This way, the logo could be larger without blocking the view of the background video.

Changes to location_template.html: Instead of making a template we ended up just starting on one of the countries and copy the HTML into the other pages since the only difference in them are the text and picture. There are also no picture in the brief description of the locations located on the bottom of the pages, as we felt it was disturbing and didn’t fit in.

Added: test.php
A "secret" script, which is used for convenience during development. Using this script, developers can copy and paste header and footer into their HTML, to see what their HTML looks like locally (without the need to install PHP on their computers). The copy-pasted material is neither used nor required when accessing the site through PHP, so modularity is ensured. See the modularity section for details on this.

Modularity

PHP is used to:

  1. perform "URL rewriting", so that the URLs look nicer (with no .html ending). The site should be accessed through "site.php", and the actual HTML files are located in a folder "content", which users are not supposed to access directly.
  2. automatically generate the menu, so that it isn't written by hand. Among other things, this enables us to use relative links while still being modular.
  3. include header and footer on every page, so that any changes to those only need to be done in one place.

More precisely, PHP looks at the HTML files in the content directory, and puts the <main> element and its contents between the header and the footer. This means that everything in the HTML file that isn't inside <main> is discarded, enabling developers to copy and paste headers and footers into their HTML files (to preview how it looks) without sacrificing modularity (since the copy+pasted material is discarded when accessed through PHP).

A positive side effect of using PHP instead of JavaScript, is that search engines will be able to see the menus, which they cannot if the menus are created with JavaScripts (since we cannot assume Google runs JavaScript on their spiders). Thus, the site is easier to find through Google (if we had decided to tell Google about it, which we won't since it's a site for a company which doesn't exist).
